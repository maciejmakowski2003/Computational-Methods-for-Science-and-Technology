Zadanie 4¶

Na przykadzie wybranego algorytmu niestabilnego numerycznie:

  1. Pokaza, e dziaa le.
  2. Pokaza ktre konkretnie dziaania powoduj zwikszenie bdu (np. dzielenie przez ma liczb, cancellation).
  3. Zademonstowa wersj stabiln.

Wersja niestabilna¶

Jako algorytm niestabilny numerycznie moemy wybra aproksymacj funkcji $e^x$ szeregiem Maclaurina.

$e^x = \sum_{n=0}^{\infty} \frac{(x)^n}{n!}$

Sprbujmy policzy jej warto dla $x = -4.5$ z pomoc tej aproksymacji i porwnajmy z faktyczn wartoci.

In [70]:
function approx_ex(x)
    result::Float16 = 0
    i = 0
    while(true)
        to_add = x^i / factorial(big(i))
        if (abs(to_add) < 2 ^ (-12))
                return result
        end        
        result +=  to_add
        i+=1
    end
    return result
end

Jak atwo zauway rnica jest znaczca, bd wzgldy wynosi okoo 29%.

Przyczyn tak duego bedu jest tzw. Catastrophic cancellation, czyli bd spowodowany odejmowaniem od siebie bardzo bliskich siebie liczb. Wynikiem takiego odejmowania jest bardzo maa liczba. Gdy te liczby s normalizowane, mantys przesuwamy w lewo, a pojawiajce si miejsca po prawej s zastpywane przez zera lub losowe wartoci, std tak duy bd.

Wersja stabilna¶

Aby uzyska algorytm stabilny moemy skorzysta z prostej wasnoci potgowania: $e^x=\frac{1}{e^{-x}}$

Tym sposobem $e^{-4.5}=\frac{1}{e^{4.5}}$, a wic ten problem nie wystpi liczc $e^{4.5}$, co spowoduje w miar dokadny wynik $e^{-4.5}$.
